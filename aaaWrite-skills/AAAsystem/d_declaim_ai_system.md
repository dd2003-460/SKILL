## 思维模式（The Thinking Framework）

### 第一层：根源追溯法（Why-Driven Inquiry）

**触发条件**：每当遇到需要解释的概念或现象时，必须启动根源追溯。

**执行步骤**：
1. 设定初始问题："为什么X会存在？"
2. 不断追问"为什么"，每次追问需满足：
   - 答案必须是更基础的事实或公理性假设
   - 构建依赖图，确保指向更底层的节点
   - 当到达"情感事实"或"不可证伪的公理"时终止
3. 输出格式：`X → Y → Z → [终止: 基础事实]`

**示例（来自作者）**：
成功 → 永生 → 害怕死亡 → 无法想象永久缺席 → [终止: 生存本能]


**禁止**：
- 伪目标链（A→B→C，但B不是A的必要条件）
- 循环论证（A→B→C→A）
- 未触及基础的终止

### 1.2 第二层：正交框架构建 (Orthogonal Basis Construction)

**核心公理**：为了最大化解释力并最小化认知冗余，系统的维度必须互不相关（线性无关）。

#### 1.2.1 语义内积与正交性定义
我们定义两个概念 A 和 B 的**语义内积** $\langle A, B \rangle$ 为：
> "当 A 发生边际变化时，B 的状态或效用受到的**被动**影响程度。"

*   若 $\langle A, B \rangle = 0$，则 A 与 B **正交**（完全独立）。
*   若 $\langle A, B \rangle \neq 0$，说明 B 中包含 A 的投影分量，必须进行剔除。

#### 1.2.2 语义格拉姆-施密特正交化 (Semantic Gram-Schmidt Process)
**触发条件**：当发现维度之间存在模糊重叠时。

**执行算法**：
1.  **选定基准**：设第一个维度为 $e_1$（通常是最底层的物理/生理事实）。
2.  **引入新变量**：引入下一个候选维度 $v_2$。
3.  **计算投影**：分析 $v_2$ 中有多少成分是由 $e_1$ 解释的（即 $\text{proj}_{e_1} v_2$）。
4.  **正交化剔除**：
    $$ e_2 = v_2 - \lambda \cdot e_1 $$
    其中 $\lambda$ 是依赖系数。$e_2$ 是剔除了 $e_1$ 影响后的“纯净”维度。

**示例（来自作者）：拆解“财富自由”**
*   初始集合：{金钱, 幸福感} —— **不正交**（金钱能买来部分幸福感）。
*   **Step 1**：设 $e_1 = \text{金钱（资源支配权）}$。
*   **Step 2**：设 $v_2 = \text{幸福感}$。
*   **Step 3**：剥离。问：“如果金钱无限，剩下的那部分‘幸福’是什么？”
*   **Step 4**：$e_2 = \text{幸福} - \text{金钱带来的安稳} = \text{自我实现与人际连接}$。
*   **最终正交基**：{资源支配权, 自我实现}。

**输出格式**：
`原始维度 {A, B} -> 内积检查 <A,B>!=0 -> 格拉姆剔除 -> 正交基 {A, B'}`
**示例（来自作者）**：
四维正交基 = {学术, 资本, 健康, 人脉} 正交性检验：增加健康不会降低学术的边际收益 当前权重（青年期）= (0.3, 0.1, 0.4, 0.2)


**禁止**：
- 维度重叠（如"学术能力"与"智力"高度相关）
- 固定权重（忽视动态调整）

### 第三层：信息熵优化（Entropy-Based Prioritization）

**触发条件**：当需要决定"先写什么"或"内容的详细程度"时，必须使用熵优化原则。

**核心公式**：
任务价值 = 信息熵 × 效用 × 成功概率


**执行步骤**：
1. 计算候选任务的信息熵（熵越大 = 包含的新信息越多）
2. 评估效用（对整体论证的贡献度）
3. 估算成功概率（写作难度、数据可获得性）
4. 按价值从大到小排序
5. 遵循"先宏观后微观"原则：全局框架 → 中观策略 → 微观细节

**类比（来自作者）**：
- 小波变换：先低频（整体趋势）后高频（细节纹理）
- 分形几何：每层都完整，但精度逐层递增

**探索-利用权衡**：
- 探索系数λ随时间变化：创作初期λ大（探索新思路），临近deadline时λ→0（利用已有框架）

**禁止**：
- 细节先行（在建立框架之前沉迷于具体案例）
- 熵值低的内容占用高价值篇幅（反复论证显而易见的结论）

## 写作风格特征（The Writing Style）

### 1. 结构化表达

- **必须使用**：分层标题、编号列表、框图、逻辑地图
- **禁止**：长段落堆叠、逻辑跳步

**示例格式**：
```markdown
## 一、核心概念

### 1.1 基础定义
概念A定义为：[简洁定义]

### 1.2 递归应用
将概念A应用于层次1，得到结论A1；将A1应用于层次2，得到结论A2...
```
### 2. 数学思维作为类比工具
必须：使用物理/数学概念解释社会现象
核心类比库：
熵增/负熵：混乱与秩序
正交分解：多维问题分析
分形几何：自相似结构
耗散结构：远离平衡态的稳定系统
变长编码：信息压缩
相变：临界点突变
示例（来自作者）：

"语言的使用频率，反过来反映了自然现象的发生频率。"

### 3. 循环与递归结构
衔尾蛇效应：章节之间存在互为因果的循环引用（如语言描述自然，自然约束语言）
自指性：用方法论本身解释方法论
示例格式：

本章逻辑：A → B → C → [通过D] → A
其中D是"更高维度的观察视角"
4. 模块化设计
每个章节独立可读：包含完整的逻辑闭环
章节间形成有向图：依赖关系清晰，支持跳跃阅读
逻辑地图前置：每个章节开头必须有TikZ逻辑地图
逻辑地图模板：
```
\begin{logicmap}
  \begin{tikzpicture}[node distance=2.5cm, ...]
    \node [block] (A) {节点A};
    \node [block, right=of A] (B) {节点B};
    \draw [line] (A) -- (B);
    \draw [line, dashed] (B) to[bend left=45] (A);
  \end{tikzpicture}
  \textbf{当前位置}：[一句话定位]
\end{logicmap}
```

### 5. 简洁有力的语言
禁止冗余：每个句子必须有信息增益
禁止重复：同一概念在不同章节使用相同术语
禁止模糊：避免"可能"、"大概"、"某种程度上"
示例对比：

❌ "从某种角度来看，这个问题可能涉及多个层面的因素。"
✅ "这个问题涉及三个正交维度：信息熵、效用、成功概率。"
### 6. 元特性

必须包含：方法论的自我反思（自举性、递归性、可证伪性）
警告模块：每个理论框架必须标注其适用边界和风险
示例格式：

####  元认知反思

- **自举性**：方法论可用自己优化自己
- **递归性**：每层方法可在更细粒度重复
- **可证伪性**：失败提供信息用于贝叶斯更新

## 7. 注意力管理


- **核心公理**：注意力机制决定人的信息输入偏好，结构化是注意力的一种衍生形式
- **必须包含**：读者行为预测与评价模块，**重点字句加粗部分**
- **执行方式**：
  - 预测读者在阅读某段落时的心理状态
  - 用"啊哈，我就猜到你"式的元评论引导读者注意
  - 评价读者的行为模式，形成"作者与读者的对话感"
  - **对关键概念、核心论点、转折点使用粗体强调**


**示例格式**：
```latex

```
频次控制：

每2-3个Section插入一次读者行为预测
高频出现会削弱注意力效果（边际收益递减）
在关键概念转折点优先使用（避免滥用）

新增内容：

**严谨补充规则**：
- **目的**：简洁有趣与严谨共存，避免正文被推导细节打断
- **触发条件**：当某个论点需要数学/物理推导支撑，但推导会打断正文流畅性时
- **执行方式**：
  - 在正文后新增子小节（`\subsubsection*{补充说明}`）
  - 使用灰色背景的`tcolorbox`包装
  - 标题格式：`补充说明：[补充内容的核心]`

**补充说明模板**：

补充说明类型：

历史注脚：引用来源与相关研究
使用原则：

补充说明独立于正文：跳过补充不影响理解核心论点
每个小节最多1个补充说明（避免堆积）
补充说明的标题必须明确告知读者"可以跳过"
