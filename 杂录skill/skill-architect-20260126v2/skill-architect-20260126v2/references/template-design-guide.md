# Design 阶段用户指南

## 设计思路

完整的设计文档模板，覆盖从需求到部署的所有关键决策。**文件变更清单**是最关键部分，必须明确列出所有将要修改的文件及行号。可选章节（测试、可观测性、安全、部署）仅在与本次变更相关时填写。

## 用户检查点

在推进到 Phase 4（Implement）前，请确认：

- [ ] 设计决策部分说明了"为什么这样做"和"为什么不那样做"
- [ ] **文件变更清单明确列出了所有将要修改的文件及行号范围**
- [ ] 实施计划分阶段且可执行
- [ ] 已评估潜在风险并有缓解措施
- [ ] 可选章节（测试、可观测性、安全、部署）根据实际需要填写或留空
